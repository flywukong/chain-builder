动画设计规格

  • UI圆形铁轨，21 个 validator 节点均匀分布在环上，每个节点展示短名称 + 版本号
  • RT当前 proposer 高亮（金色脉冲光环），新块产出时沿轨道触发一个光点向顺时针方向滑动
  • UI光点颜色 = builder 颜色（blockrazor=金色、puissant=紫色等），MEV 块光点更亮
  • RT环心显示：当前块高 / Block Time / MEV% 三行实时数据
  • UI慢块 / 超时块时，对应 validator 节点短暂变红并震动
  • NEWSlash 事件触发全局警报动效：红色脉冲从受影响节点扩散

数据来源映射

  • RPCparlia_getTurnLength → 当前 turn 的 validator 顺序
  • RPCeth_getBlockMevInfo → builder / 是否 MEV
  • RPCmilliTimestamp diff → 精确 block time (ms)
  • CONSlashIndicator.getSlashIndicator() → slash 状态
  • KETnode_stats{NodeType} → 版本号显示在节点旁

交互设计

  • UXHover 节点 → tooltip 展示：全地址 / 版本 / slash count / 最近出块时间
  • UXClick 节点 → 侧滑面板展示该 validator 的详细历史出块记录
  • AI右键节点 → "AI 分析该 Validator" → 调用 Claude 生成运行状态摘要

补充建议 — 让这个工具真正覆盖研发日常

⚡链上事件时间轴

把 Slash / 大流量 / 慢块 / Reorg 统一放在一条时间轴上，任意缩放。配合 AI 分析按钮，点一个事件段就能生成该时间窗口内的完整分析报告。

🔬升级版本追踪

实时展示各 validator 运行的 BSC 版本，自动识别哪些节点还在跑旧版、距当前 release 落后多少个版本，支持一键生成升级提醒。

🏗Fast Finality 监控

追踪 parlia_getJustifiedNumber，展示 justified lag（当前块 vs 已 justified 块高度差），异常时触发告警，对 BEP 快速确认场景尤其关键。

📦合约部署跟踪

监控系统合约（ValidatorSet / StakeHub 等）的 upgrade 事件，自动 diff ABI 变化，标注哪个 PR / commit 触发了链上变更。

🧪一键链上复现

针对异常块/交易，提供"在本地 fork 复现"按钮，自动生成 hardhat/foundry fork 命令，带上精确块高和 tx hash，减少开发者手动查询时间。

📋值班日报生成

每日自动汇总：异常块次数、Slash 情况、磁盘告警、Issue 新增数量、Bug Bounty 状态，AI 生成一段值班摘要，一键复制发到群里。
